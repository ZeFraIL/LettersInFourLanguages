# Подробное описание класса MainActivity

Этот документ поможет тебе разобраться, как устроена логика главного экрана нашего приложения. Мы разберем всё: от переменных до жизненного цикла.

---

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность).
    *   *Термин:* **Activity** — это основной компонент Android-приложения, который представляет собой один экран с пользовательским интерфейсом.
*   **Назначение:** Это главный и единственный экран приложения. Он отвечает за выбор языка, генерацию случайной буквы и анализ её свойств (соседи, позиция, регистр).
*   **Взаимодействие:** Взаимодействует с файлом разметки `activity_main.xml` для отображения элементов управления и вывода результата.

---

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `randomLetter` | `char` | Хранит последнюю сгенерированную случайную букву. | В методе `onCreate` при нажатии на кнопку. |
| `englishAlphabet` | `char[]` | Массив символов английского алфавита. | Для выбора случайной буквы и поиска позиции/соседей. |
| `russianAlphabet` | `char[]` | Массив символов русского алфавита. | Аналогично английскому. |
| `georgianAlphabet` | `char[]` | Массив символов грузинского алфавита. | Аналогично английскому. |
| `amharicAlphabet` | `char[]` | Массив символов амхарского алфавита. | Аналогично английскому. |

*   *Термин:* **char** — тип данных для хранения одного символа (буквы, цифры или знака).
*   *Термин:* **char[]** — массив (список) символов.

## 3. Методы класса

### Метод: `onCreate`
*   **Тип метода:** `protected` (защищенный, доступен внутри пакета и наследникам).
*   **Возвращаемое значение:** `void` (ничего не возвращает).
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Объект с данными для восстановления состояния экрана после поворота или закрытия. |

*   **Что делает метод:**
    1. Вызывает базовую реализацию `super.onCreate`.
    2. Устанавливает внешний вид экрана через `setContentView(R.layout.activity_main)`.
    3. Находит все элементы управления (кнопки, переключатели, текстовые поля) по их ID.
    4. Устанавливает "слушатель" (`OnClickListener`) на кнопку "Analyze".
    5. Внутри слушателя: проверяет, какой язык выбран, генерирует букву, проверяет галочки (CheckBox) и формирует итоговый текст.

*   **Когда вызывается:** Автоматически системой Android при создании экрана (запуске приложения).

---

### Метод: `generateRandomLetter`
*   **Тип метода:** `private` (личный, доступен только внутри этого класса).
*   **Возвращаемое значение:** `char` (случайная буква).
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `alphabet` | `char[]` | Массив букв выбранного языка. |

*   **Что делает метод:** Использует класс `Random`, чтобы выбрать случайный индекс (номер позиции) в массиве и возвращает букву, которая там находится.
*   **Когда вызывается:** Внутри обработки нажатия кнопки, когда нужно получить новую букву.

---

### Метод: `getNeighbor`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `char` (соседняя буква или символ '-')
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `letter` | `char` | Текущая буква. |
    | `alphabet` | `char[]` | Массив алфавита. |
    | `offset` | `int` | Смещение (-1 для предыдущей, 1 для следующей). |

*   **Что делает метод:** Находит текущую позицию буквы, прибавляет к ней смещение и возвращает букву по новому адресу. Если мы выходим за границы алфавита (например, ищем соседа для первой или последней буквы), возвращает прочерк.
*   **Когда вызывается:** Когда пользователь отметил галочку "Show adjacent letters".

---

### Метод: `getAlphabetPosition`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `int` (номер буквы в алфавите от 1 до N).
*   **Параметры:** `letter` (буква), `alphabet` (массив).
*   **Что делает метод:** Пробегает по всему массиву циклом `for`. Как только находит совпадение, возвращает индекс + 1. Если не нашел — возвращает -1.

---

## 4. Жизненный цикл (Activity)
В данном классе явно переопределен только один метод:
*   **`onCreate()`**: Вызывается в самом начале. Здесь происходит вся настройка: создание интерфейса и логики кнопок. Без этого метода приложение просто не знало бы, что рисовать на экране.

---

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:**
    *   `RadioGroup (rgLanguages)`: Группа кнопок для выбора одного языка из списка.
    *   `CheckBox`: Галочки для включения доп. функций (соседи, позиция, регистр).
    *   `Button (btnAnalyze)`: Кнопка, запускающая расчет.
    *   `TextView (tvResult)`: Поле для вывода результата крупным шрифтом.
*   **Связь с кодом:** Используется метод `findViewById(R.id.имя_элемента)`. Это классический способ "связать" XML-разметку с Java-объектом.

---

## 6. Взаимодействие с другими компонентами
*   В данном коде переходы (`Intent`) или работа с БД отсутствуют. Приложение полностью автономно и работает в рамках одного экрана.

---

## 7. Общая логика класса
1. Пользователь выбирает настройки на экране.
2. При нажатии кнопки программа смотрит на `RadioGroup` и определяет, какой массив алфавита использовать.
3. Выбирается случайная буква.
4. Если стоят галочки, программа дописывает в строку `result` информацию о соседях, номере или меняет регистр.
5. Результат выводится в `TextView`.

---

## 8. Объяснение простыми словами
Представь, что **MainActivity** — это ведущий викторины.
1. У него в руках несколько мешочков с буквами (английский, русский и т.д.).
2. Ты (пользователь) говоришь ему: "Вытяни букву из русского мешочка и скажи, кто её соседи".
3. Ведущий заглядывает в мешочек (`alphabet`), достает случайную бумажку (`generateRandomLetter`).
4. Затем он сверяется со списком, чтобы найти, кто стоит до и после этой буквы (`getNeighbor`).
5. В конце он записывает всё на большой доске (`tvResult`), чтобы ты увидел результат.

## Заметки по коду (Советы по улучшению)
*   **Дублирование кода:** В `onCreate` генерация буквы прописана внутри каждого `if`. Можно было сначала выбрать массив, а потом один раз вызвать `generateRandomLetter(alphabet)`. Это сделало бы код короче.
*   **Локализация:** Тексты ("Select language!", "Random letter:") прописаны прямо в коде (хардкод). Правильнее выносить их в `strings.xml`.
*   **Грузинский и Амхарский алфавиты:** Для этих языков смена регистра (`Character.toLowerCase/toUpperCase`) может работать не так, как ожидается, или не работать вовсе, так как в этих языках нет понятия "большая/маленькая буква" в классическом понимании.
